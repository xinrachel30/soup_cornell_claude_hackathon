# soup_cornell_claude_hackathon

## Inspiration
The desire to help remove decision fatigue from the modern person's life.
## What it does
Helps aggregate courses from a variety of sources so that users can find the relevant options, quickly.
## How we built it
We used React, Typescript, and FastAPI for the structure. We used Gemini API to help interpret user reviews. We tried a variety of scraping tools, including the DuckDuckGo library, BeautifulSoup, and others.
## Challenges we ran into
The main challenge we ran into was in terms of data integrity, and finding the best libraries to use.
## Accomplishments that we're proud of
We are particularly proud of navigating the many tools available and being able to eventually find some authentic user reviews to include on our web app.
## What we learned
We learned the value of human interpretation, which ultimately guides progress and next steps. LLMs help a lot with structure and framing, and this makes context much more valuable.
## What's next for Course Aggregator
For next steps, we hope to find more reliable user reviews, figure out how to develop better metrics to display to the user, and formalize the data scraping pipeline.
